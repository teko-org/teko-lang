1. WHAT HAS BEEN DELIVERED, TESTED AND CERTIFIED (CODE HISTORY)
-------------------------------------------------------------------------------
* MULTI-ENGINE ARCHITECTURE (Phase 5): The entire Ahead-of-Time (AOT) backend
  was segregated into subfolders by native OS/CPU (src/codegen/apple, linux,
  windows, bsd_unix, bare_metal). Clean 3-parameter signatures (ctx, op, arg),
  strict C23, eliminating magic numbers via the OP_PROLOG and OP_EPILOG enums.

* FINAL ORCHESTRATOR WITH TRIPLE OPTIMIZATION (Phase 6): The central file
  `src/codegen/codegen_metal.c` unified the following layers in batch:
    - Dead Code Elimination (DCE): Removes instructions after OP_RETURN/OP_HALT,
      reactivating flow only via Label markers (op >= 100). The dead-zone trigger
      fires after emitting the termination instruction, ensuring machine-halt
      kernel interrupts are always emitted.
    - Common Subexpression Elimination (CSE): Purges consecutive redundant opcodes
      and loads (e.g. identical OP_ICONST entries).
    - Constant Folding: Local predictive scan that intercepts mathematical
      expressions of static literals at compile time (e.g. 20 + 30) and collapses
      the result into a single OP_ICONST 50. Uses the neutral byte 0xCC to clear
      the redundant bytes, protecting the 0x00 of the legitimate OP_HALT.

* NATIVE STATIC LINKER ENGINE (Phase 7 - Complete and Polymorphic):
    - `tld_elf.c`/`.h`: Direct structural byte-writing of 64- and 32-bit headers
       (ELF64/ELF32) for x86, ARM, RISC-V, MIPS, and PPC64. Injects the static
       string section `.rodata` and the UNIX structural note tag (`.note.tag` with
       EI_OSABI = 9) for native FreeBSD certification.
    - `tld_macho.c`/`.h`: Writing of executables and dynamic libraries (.dylib)
       for Apple macOS. Injects `LC_MAIN` and `LC_BUILD_VERSION` load commands,
       applying the `macho_align_to_page` math macro for strict hardware page
       alignment (4KB Intel / 16KB ARM64).
    - `tld_pe.c`/`.h`: Writing of Microsoft PE/COFF headers (Windows x86,
       x86_64, ARM64). Controls injection of the `PE_CHAR_DLL` flag to generate
       DLLs, and performs padding via zeroed buffers respecting the disk alignment
       `FileAlignment` (multiples of 512 bytes).
    - `tld_elf_reloc.c`: Relocation Engine computing relative jump offsets and
       absolute patches in Little Endian.
    - `tld_elf_arch.c`: Complete dictionary of machine hexadecimal opcodes.
    - `tld_wasm.c`: Binary WebAssembly emitter (.wasm) with LEB128 compression.

* THE CORE NATIVE EMBEDDED RUNTIME (Phase 8 - Validated):
    - `teko_mem_sys.c`: Requests and releases raw virtual memory pages directly
       from the kernel via pure syscalls (mmap on Unix, VirtualAlloc on Win).
    - `teko_thread.c`: Cooperative M:N Round-Robin scheduler with 64KB stacks
       isolated per Green Thread and simulated context switching.
    - `teko_channel.c`: Synchronous channels with blocking/unblocking queues.
    - `teko_arena.c`: Region Manager controlling 128KB local Sub-Arenas per
       thread. O(1) allocation with 8-byte alignment and 1-clock recycling.

* CENTRAL MULTI-MODE CLI DRIVER (`src/main.c`):
    - Orchestrates the batch pipeline, allowing interruptions via flags: `-S` to
      stop at auditable assembly text, `-c` to stop at raw binary object, or to
      compile and link third-party textual `.s` files directly.

2. MAP OF PHILOSOPHICAL AND CONCEPTUAL REQUIREMENTS (THE LONG-TERM VISION)
-------------------------------------------------------------------------------
* NATIVE TEST STRUCTURING AND COVERAGE (.tkt):
    - `.tkt` extension for co-located test files (same tree as the tested object).
      Release builds automatically ignore these files.
    - Native Code Coverage via compile-time-assisted instrumentation at Codegen,
      injecting counters into RAM at the start of each Basic Block. The linker
      will embed the `.teko_cov_map` section associating counters with source
      code lines. The runtime will flush counters at process end in binary format
      (.tkcov).

* NATIVE LITERAL SUFFIXES (Literal Add-ons):
    - Captured in the Lexer with zero runtime cost: Time (ms, s, m, h, d), Data
      (b, kb, mb, gb), and Socket Bandwidth (kbps, mbps, gbps).

* INTEROPERABILITY AND THE `.teko_meta` SECTION:
    - Resolution via `include_paths`, `static_links`, and `dynamic_links` in .tkp.
    - Teko modules embed rich type metadata in the `.teko_meta` section.
    - Pure C objects expose signatures via automatic header (.h) parsing.
    - Managed runtimes (.NET non-AOT, JVM) are isolated and handled via IPC.
      Native static support guaranteed when .NET uses Native AOT compilation.

* COMPREHENSIVE NETWORKING (OSI LAYER 4 TO LAYER 7) AND CRYPTOGRAPHY:
    - Raw sockets, TCP, UDP, and asynchronous QUIC via io_uring/kqueue/IOCP.
    - Native TLS 1.3 embedded in the IP bus.
    - Integrated polyglot routing and handling covering HTTP/1.x, HTTP/2,
      HTTP/3, HTTP/4, bidirectional WebSockets (ws_chan), and gRPC (RPC).
    - Hardware-accelerated cryptography (AES-NI/AVX): SHA256, SHA512, BLAKE3,
      AES256-GCM, CHACHA20-POLY1305, ED25519, and RSA.

* ENTERPRISE PARSERS AND EMBEDDED TEMPLATE COMPILER:
    - O(1) linear execution free of reflection: `parse.json`, `parse.csv`, `parse.xml`.
    - Native template engine integrated via rich String Literals: `html"""..."""`.
    - Bundler and Minifier integrated at compile time: `bundle()` and `minify`
      commands optimize and embed static CSS/JS/Assets into the `.rodata` section.

* ADVANCED CONCURRENCY PRIMITIVES, SIGNALING, AND CORE DUPLEX CHANNELS:
    - `routines`: Fires pure background tasks and executions at the runtime level.
    - `duplex chan`: Native Full-Duplex channels (symmetric bidirectional) that
       manage isolated RX and TX buses at the hardware level. Features an internal
       state machine capable of signaling legitimate closure (`.close()`) or
       drops/failures. Unblocks and wakes blocked threads on crashes, returning
       structured errors. Can be used as an alternative, safe consumption model
       for isolated dependencies without export.
    - `delayed chan`: Timed channels. Messages receive timestamps and consuming
       threads are suspended in the Timer Queue, woken by interrupts.
    - `broadcast chan`: Non-destructive 1:N Pub-Sub channels based on registers.
    - Automated Shared Memory: `shared` block and `atomic` control. The compiler
      transparently injects lightweight locks (Spinlocks/Memory Fences).
    - `circuit` (Circuit Breaker) coupled to `retry` routines:
        - Support for `exponential` and `logarithmic` backoff algorithms.
        - Can be limited by `attempts`, global `timeout`, or both combined.
        - If both limits are provided, the compiler calculates the incremental
          relative retry time, jumping directly to the `fallback` if the time
          limit is exceeded.
    - Native Web Architecture via Keywords: Expressive syntax for APIs and
      micro-applications (`api`, `middleware`, `get`, `post`, `put`, `delete`,
      `rpc`, `websocket`, `use`). AOT-compiled static Radix tree generation.

* BARE-METAL OBJECT-ORIENTED PARADIGM:
    - Support for Concrete, Generic (<T> via monomorphization), and Abstract classes.
    - Complete rejection of runtime object Attributes/Annotations.
    - Multiple behavioral inheritance implemented via `traits`.
    - Event Subsystem (`event`, `raise`, `subscribe`): Control defined at
      subscription: `fanout` (parallel Green Threads) or `fire_and_forget`.

* ZERO-OVERHEAD OPTIONALS AND COMPILED METAPROGRAMMING:
    - `?T` nullability via compacted Value Types. The Elvis operator (`??`) compiles
      directly to hardware conditional instructions (`je`/`cbz`).
    - `comptime`: Code execution at build time. Metaprogramming occurs at compile time.
    - `soa` (Structure of Arrays): Optimization that reorganizes vector layouts in
      RAM to invisibly enable pure native SIMD vectorization.
    - `defer`: Syntactic registration of mandatory scope-closing routines.
