Teko · memory-model research · surface study

Six theoretical memory models, and what they would actually look like in your hands

A scenario-by-scenario look at how each candidate model would appear on the surface of Teko — measured against today's arena. The headline finding is not about power. It is about visibility: half of these models have no surface syntax at all, which is precisely what makes them the low-risk ideas.

01  The split that matters

Which of these you can even see

Before comparing capability, sort the models by whether the programmer ever writes anything. This is the load-bearing distinction: a model with no surface can only ever tighten an existing rule or optimise a runtime, never regress a working program — so its cost of being wrong is near zero. That is the whole reason to reach for it first.

Real surface

You write a sigil or keyword

A visible construct enters the source. This is where the ergonomic cost — and the language bet — lives.

NOEMA (&T read-borrow) · Relevant types (#must_free) · Adoption & abandon (region / adopt / abandon)

Invisible — internal

A proof lens on the IR

No keyword, no qualifier, nothing at the source. The model's entire effect lives in the compiler's own conscience.

Ordered types — the LIFO fragment as an internal soundness proof over the region tree.

Mostly inferred

The compiler fills it in

A qualifier exists in principle but you almost never write it; it is inferred from analysis Teko already runs, then erased.

Second-class values (@local / may-not-escape) — the pilotable idea.

Runtime only

An allocator swap

Not a language feature at all — a teko_rt.c change, optionally surfaced as one scope beside defer.

REAPS — recycle freed blocks back into the bump path.

Read this as the thesis of the whole page. Only three of the six put anything in front of the programmer. The other three — ordered types (internal proof), REAPS (runtime allocator), and second-class values (inferred) — are, from the keyboard, indistinguishable from today's Teko. "Has no surface" is not a footnote; it is the finding. It is what sorts the safe experiments from the language bets.

02  Per-model profile

What it feels like & what it actually buys Teko

Each model in one honest band: the felt experience at the keyboard, and the single thing it genuinely adds — stated without overselling. Expand any card for the full read.

Second-class values

Pile A · steal a piece Surface: mostly inferred

What it feels like

Programming feels exactly like today's Teko 99% of the time — you never write the qualifier — right up until the compiler stops you at a return or a field-store: "this Ref is second-class, it cannot leave here." It is the current informal Ref-can't-escape rule turned into a checker that actually catches you. The language feels more honest, not heavier.

What it actually buys Teko

A sound, inferable foundation under the ad-hoc "Ref may not escape a field" rule Teko already enforces informally. The compiler infers a @local bit per binding from the escape analysis it already runs, checks it at every escape boundary, and erases it before codegen — pure C stack locals, zero runtime cost. It buys soundness + inference, not new capability: zero power over escaping, cyclic, or shared data. Its whole value is that it changes almost nothing observable. The honest gate: "just infer it, zero annotation" is contingent on auditing that Teko's name-based escape analysis is actually sound for aliased/returned values — that audit is the real cost, not the qualifier. It leaves every hard cliff (graph-leak, aliased-free, cycles, concurrency) exactly where it was.

Ordered types — LIFO fragment of ordered logic

Pile A · steal a piece Surface: invisible / internal

What it feels like

It feels exactly like current arena Teko — because it is current arena Teko. Nothing changes in your hands. The only difference lives in the compiler's conscience: what used to be a hand-audited belief ("the region tree is stack-shaped, so drops are safe") becomes a discharged proof obligation on the IR. You never see it, never write it, never fight it.

What it actually buys Teko

A type-theoretic name and soundness proof for the region tree Teko already ships — nothing more, and that is the point. Ordered logic (Polakow, CMU 2001) is the LIFO fragment of linear logic: later-introduced resources are consumed before earlier ones — precisely the frame/loop/arm region tree. It turns arena correctness from hand-audit into sound-by-construction, finding exactly where the current per-binding-name escape analysis is unsound. But it is honest about its ceiling: pure ordered types express strictly less than Teko already does — strict LIFO forbids any non-nested lifetime, so S2 return-fresh and the promote-to-root path are, in the pure calculus, ill-typed. That refusal is the diagnostic value: it maps exactly where the arena's safety rests on machinery beyond the ordered fragment.

NOEMA — Neut-style &T read-borrow

Pile A · steal a piece Surface: real syntax

What it feels like

Almost identical to Ref-based Teko, except read-only access gets a distinct sigil: any function that only observes a value takes it as &T, and the compiler forbids that borrow from being stashed in a field, returned, or aliased past its source. The mental model is "reads are cheap, tethered, and can never be the pointer that dangles"; anything that must outlive or own still uses ordinary values / Ref.

What it actually buys Teko

Exactly one thing, cleanly: it makes the read side of the aliased-free cliff (S6) sound by construction. A &T noema is statically forbidden from being stored or outliving its source, so it can never become the surviving alias that dangles after a free. Zero runtime cost — the sigil erases to a plain C pointer. What it does not buy: no new capability — it cannot host cycles, reclaim graphs, or make owning aliases sound. Critically, the rule is extracted from a viral full-linearity discipline Teko refuses to adopt, so "the &t rule alone is sound in a non-linear language" is unproven engineering: Teko must itself prove its name-based check enforces the no-store/no-escape invariant the noema requires.

Relevant types — #must_free must-use handle

Pile A · steal a piece Surface: real syntax

What it feels like

Like the compiler handing you a receipt you are legally required to spend: every relevant handle nags at you until you free, return, or move it, so leaks turn from invisible drift into a red squiggle you clear on the same line you allocate. The discipline is entirely about not-forgetting-to-free — and it stays conspicuously silent the moment two names point at one buffer.

What it actually buys Teko

A single, opt-in, fully-erasable annotation that converts Teko's most common silent failure — leak-to-root in a long-running program — into a compile error, at zero runtime cost and with no new subsystem. It rides entirely on the arena + existing teko::mem::free, adding only a consume-or-fail dataflow check. Genuinely valuable exactly where leaks hurt (S2 escaping allocs, S5 caches) and honest about buying nothing on the hard cliffs: for cycles/DLLs it collapses to "the region must be closed" (which lexical arenas already guarantee), and for aliased-free it is powerless by construction. Being the relevant / must-consume dual, it enforces at-least-once freeing and has no notion of uniqueness — so it can make a use-after-free more likely while eliminating the leak.

REAPS — region allocator with recycling free

Pile A · steal a piece Surface: runtime only

What it feels like

Like nothing changed — you write the same arena Teko you already write, and 99% of the time the allocator swap under teko_rt.c is invisible. The one moment it surfaces is wrapping a churny, phase-structured cache in a reap { } block that reads exactly like the defer / arena_push you already know, quietly keeping that phase's footprint from creeping upward.

What it actually buys Teko

A footprint plateau for churn-heavy, phase-structured workloads (servers, caches, memo tables) at near-zero surface cost — recycle-freed-blocks-into-the-bump-path under teko_rt.c. The honest ledger is thin: it plateaus fragmentation and does not compact; Teko already ships a weaker version (park-on-freelist), so the real delta is just coalescing + feeding runs back to the bump pointer. And it is safety-neutral-to-negative: no reachability fact, so it cannot touch any correctness cliff, and by recycling freed memory sooner it makes the S6 use-after-free more likely to return corrupted bytes. Buy it as a runtime footprint optimisation for the eventual server era, not as an answer to any correctness cliff.

Adoption & abandon — Mezzo adopter regions

Pile B · bet the language Surface: real syntax

What it feels like

Like drawing a box around a knot of mutually-pointing objects and saying "these are one thing now": inside the box you point wherever you like — cycles, back-edges, aliases, all fine — and you pay nothing until the moment you try to pull one object back out, where a runtime tag-check makes you prove it is really leaving. It is the arena-of-nodes + index workaround Teko already reaches for, except the region is a first-class value and the "is this index still valid" check is a language-enforced dynamic guard instead of a comment.

What it actually buys Teko

Exactly one thing the shipping arena cannot express: a sound, Coq-verified way to host a genuinely cyclic/aliased mutable graph in a bulk-reclaimed region without a per-op refcount and without a tracing GC — the S3/S4 territory. It promotes the hand-rolled arena-of-nodes pattern into a checked construct. The price and boundary, honestly: it does not compute the reachability fact Teko lacks — it sidesteps it by making the adoption tag a runtime word and abandon a runtime branch. It closes the cyclic-graph cliff but leaves aliased-free (S6) essentially untouched, and full Mezzo is the permission calculus whose own authors wrote a post-mortem — so the affordable version is a narrow library-plus-two-keywords hatch. A real capability for the graph corner, paid for in runtime tags + a per-object dynamic check. Pile B.

03  The comparison proper

Scenario by scenario — every model's surface, its delta from arena, its honest flag

Six canonical shapes. Under each, all six models, in a fixed order: the three invisible/inferred/runtime ones (second-class, ordered, REAPS) that mostly show no delta, then the three real-surface ones (noema, relevant, adoption). Watch how much of the matrix is "nothing changed" — that emptiness is the substance.

S1 Binary tree — build then sum The arena's happy path: a nested, acyclic structure read-only traversed and dropped in bulk. The baseline against which every delta is measured.
Second-class valuesinferred
/** Second-class is INFERRED — S1 surface is byte-identical to today. */
pub type Node = class {
    pub value: i64
    pub left: Node?
    pub right: Node?
    pub fn leaf(v: i64) -> Node { Node { value = v; left = null; right = null } }
    pub fn branch(v: i64, l: Node, r: Node) -> Node { Node { value = v; left = l; right = r } }
}

/** `n` is inferred @local (read-only, never stored/returned) — the qualifier rides invisibly. */
pub fn sum(n: Node) -> i64 {
    let ls = match n.left { Node as l => sum(l); _ => 0 }
    let rs = match n.right { Node as r => sum(r); _ => 0 }
    n.value + ls + rs
}

pub fn run() -> i64 {
    let l = Node::leaf(2)
    let r = Node::leaf(3)
    let root = Node::branch(1, l, r)   // inferred first-class: flows into sum, never escapes run
    sum(root)   // 6
}
Delta from arenaNothing visible. The checker infers a @local bit on n (only read) and proves root stays inside run. No annotation written; code identical to arena S1.
LoweringIdentical to today: classes → heap-arena objects, sum recurses over pointers, the qualifier erased before codegen. Zero runtime cost — the bit exists only during checking.
Honest flagThe qualifier does nothing here — pure inference on a program that already type-checks. Not evidence the model helps; evidence it is invisible in the common case. Buys nothing beyond a soundness lens.
Ordered typesinternal proof
/* Source is byte-identical to arena Teko (Node/sum/run as above).
 * INTERNAL PROOF OBLIGATION (invisible):
 *   ordered context for run()   Γ = [ l < r < root ]   (intro order = arena order)
 * The whole tree lives in run()'s frame region, consumed on frame exit in reverse:
 *   root ⊣ r ⊣ l.  Ordered typing DISCHARGES this: every alloc is freed
 *   strictly-later-introduced-first, so the single bulk drop is provably sound —
 *   sum() only READS and root outlives l,r within the SAME frame. */
Delta from arenaNothing (invisible). Byte-identical source; the only addition is an internal ordered-context annotation the compiler attaches to the IR, discharges, then erases.
LoweringExactly today's: one frame region for run(), bump-allocated Nodes, a single region drop at frame exit. Proof erased before codegen; zero runtime and zero code-size delta.
Honest flagThe only scenario where ordered types and Teko agree perfectly — and even here the lens proves nothing the arena didn't already do; it replaces "we believe the drop is safe" with "the ordered context proves it." Confidence, not capability. It does not prove the class-pointer edges are non-aliasing; it assumes the tree is acyclic.

NOEMAreal syntax
/** Tree summed through read-borrows: sum() only OBSERVES, so it takes &Node. */
pub fn sum(n: &Node) -> i64 {
    let ls = match n.left { Node as l => sum(&l); _ => 0 }
    let rs = match n.right { Node as r => sum(&r); _ => 0 }
    n.value + ls + rs
}
pub fn run() -> i64 {
    let l = Node::leaf(2); let r = Node::leaf(3)
    let root = Node::branch(1, l, r)
    sum(&root)   // 6
}
Delta from arenasum() takes &Node instead of Node; recursive calls pass &l/&r. The tree is still arena-owned; only the read traversal is a borrow. Ownership and construction unchanged.
Lowering&Node erases to a plain const Node*. No copy, no discard/free on the borrow. Identical machine code to today's by-reference class pointer; the sigil is a compile-time-only tether.
Honest flagPure cosmetics — no dangling risk here, so the noema buys nothing except signalling intent. On a well-nested recursion the "never stored/returned" check is trivially satisfied and adds no safety a plain Node param lacked.
Relevant typesreal syntax
pub fn run() -> i64 {
    #must_free let l = Node::leaf(2)
    #must_free let r = Node::leaf(3)
    #must_free let root = Node::branch(1, l, r)  // consumes l,r into the tree —
                                                 // their obligation transfers to root
    let total = sum(root)                        // 6
    teko::mem::free(root)                        // discharges the whole tree
    total
    // delete the free() line and run() would NOT compile:
    //   error: relevant handle `root` reaches end of scope unconsumed
}
Delta from arenaAdds #must_free on three bindings + an explicit free(root). Arena omits both (the tree leaks to root, reclaimed only at exit). Consuming l,r into branch transfers their obligation to root — one free, not three.
Lowering#must_free is a compile-time relevance bit, fully erased. Node allocs → identical arena bump-allocs; free(root) → the existing free-list-over-arena call. Zero runtime cost; the only artifact is a compile error on a missing free.
Honest flagCatches the leak, nothing more. After free(root) the child pointers dangle exactly as in S6 — reading them is still an un-diagnosed UAF. The obligation-transfer rule presumes Teko sees that branch stores its args — leaning on the same reachability fact it does not yet compute; the honest MVP restricts #must_free to same-scope frees.
Adoption & abandonreal syntax · Pile B
pub type Node = class in region {   // `in region` = adoptable: carries a hidden adopter tag
    pub value: i64; pub left: Node?; pub right: Node?
}
pub fn run() -> i64 {
    region arena {                              // opens an adopter region
        let l = adopt(arena, Node { value = 2; left = null; right = null })
        let r = adopt(arena, Node { value = 3; left = null; right = null })
        let root = adopt(arena, Node { value = 1; left = l; right = r })
        sum(root)   // 6
    }                                           // arena drops: whole tree freed in bulk
}
Delta from arenaAdds region arena { } and adopt(arena, …) around allocation; class marked class in region carries an adopter tag. Same result as arena S1 (which needs none of this). Pure overhead: extra syntax + a per-node tag word for zero benefit.
Loweringregion arena → bump arena push/pop (today's frame region). Each Node gains a hidden uintptr adopter field set by adopt. sum unchanged. Region drop = single reset. Tag words are dead weight here (never read).
Honest flagFor a pure tree this model buys nothing over the shipping arena and costs one tag word per node plus adopt-call noise — the arena already handles nested-lifetime trees perfectly. S1 only demonstrates the syntax; value does not appear until cycles / back-edges (S3 / S4).
S2 Return-fresh — build a list locally and return it The case the arena was built to handle via escape-analysis + promote-to-root. This is where the invisible models diverge most sharply: some name it, some reject it, one regresses it.
Second-class valuesinferred
/** THE SCENARIO THAT MATTERS. The RETURN is the decision point:
 *  `xs` MUST be inferred first-class, or the return is rejected. */
pub fn make_range(n: i64) -> []i64 {
    mut xs: []i64 = teko::list::empty()   // inference starts xs as @local (frame-scoped)
    mut i = 0
    loop {
        if i >= n { break }
        xs = teko::list::push(xs, i)
        i++
    }
    xs   // RETURN forces first-class: checker PROMOTES xs here, or errors
}

 * INTERNAL PROOF OBLIGATION (invisible) — and it FAILS in the pure calculus:

REAPSruntime
// Source byte-identical to arena S2. The list grows by repeated push:
// each push retires the OLD backing buffer. Under REAPS those retired buffers

Delta from arenaNothing visible. The runtime delta: intermediate (superseded) list buffers from each push are recycled by the allocator instead of sitting dead until frame-drop.

NOEMAreal syntax
/** RETURN-FRESH: the result ESCAPES, so it CANNOT be a noema. A &T may never be returned. */
pub fn make_range(n: i64) -> []i64 {   // owns + escapes → plain owning value, NOT &[]i64
    mut xs: []i64 = teko::list::empty()
    mut i = 0
    loop { if i >= n { break }; xs = teko::list::push(xs, i); i++ }
    xs   // escapes to caller/root region — an owning value, arena-promoted as today
}
Delta from arenaNothing changed vs the real arena version — and that is the point. A returned fresh list is exactly what a noema forbids, so make_range keeps its owning -> []i64 signature. The model draws the line here rather than adding syntax.
LoweringIdentical to today: xs promoted out of the frame into caller/root. No borrow involved, so no noema machinery participates.
Honest flagThe noema deliberately does not help return-fresh — it is a confinement rule, not an ownership-transfer mechanism. If anything it makes the escape boundary explicit (you literally cannot write &[]i64 as the return type), a small documentation win, not a capability win.
Relevant typesreal syntax
pub fn make_range(n: i64) -> #must_free []i64 {
    // Return type is a relevant handle: the CALLER inherits the free obligation.
    #must_free mut xs: []i64 = teko::list::empty()
    mut i = 0
    loop {
        if i >= n { break }
        xs = teko::list::push(xs, i)  // rebind keeps the SAME obligation on xs
        i++
    }
    xs   // RETURN discharges here, re-emits to caller via #must_free return type
}
pub fn run() -> i64 {
    #must_free let xs = make_range(4)          // obligation received from callee
    let total = xs[0] + xs[1] + xs[2] + xs[3]  // 6
    teko::mem::free(xs)                        // caller discharges it
    total
}
Delta from arenaReturn type gains #must_free; caller gains #must_free let + explicit free(xs). Arena returns plain []i64 and never frees. Returning is the sanctioned way to discharge the obligation in the callee and re-raise it on the caller.
LoweringQualifier erased; make_range → the identical escape-promotion the arena already does. free(xs) → the same mem::free call. The only compile-time addition is a dataflow check that every path frees, returns, or moves xs. No runtime delta.
Honest flag · clean caseThe clean case for a relevant type — an escaping allocation is exactly what leaks silently today, and threading the obligation through the return turns "caller forgot to free" into a compile error. Caveat: it works only because the obligation propagates through the xs = push(xs,…) rebinds as the-same-handle (a cheap, sound syntactic move rule). It still frees nothing the arena wouldn't reclaim at exit — it just makes the free mandatory and checked for long-running programs.
Adoption & abandonreal syntax · Pile B · regresses
/** RETURN-FRESH is the HARD case for adopters. A region is LEXICAL, so a node
 *  adopted into a local region cannot escape the block. To return, thread the
 *  caller's region as a parameter (or abandon + re-adopt, paying per-object checks). */
pub fn make_range(dst: region, n: i64) -> []i64 {
    mut xs: []i64 = teko::list::empty()
    mut i = 0
    loop { if i >= n { break }; xs = teko::list::push(xs, i); i++ }  // buffer adopted into dst
    xs   // escapes safely: already lives in the caller-owned region dst
}
pub fn run() -> i64 {
    region here {
        let xs = make_range(here, 4)   // built directly in `here`
        xs[0] + xs[1] + xs[2] + xs[3]  // 6
    }
}
Delta from arenaArena returns a fresh list with zero ceremony (auto-promotes to caller/root). Here the region is lexical and non-escaping, so you must thread dst: region as a parameter, or abandon-and-re-adopt across the boundary (an O(n) dynamic check per node). A regression in ergonomics vs shipping Teko for the escape case.
Loweringdst: region → passing the caller's arena pointer down; push allocates in dst's bump space so it survives the callee frame. The abandon/re-adopt alternative would incur a runtime tag-equality check + rewrite per element — O(n) purely to move ownership.
Honest flag · the model's worst caseThe honest counter-weight to the S3/S4 win: adopter regions do not auto-promote escapees, so the exact case the shipping arena is designed for becomes either region-threading boilerplate or an O(n) abandon/re-adopt tax. The frontier does not give escape for free here — it makes it harder than the arena already does.
S3 Cyclic graph — mutual back-edges, capped walk The first real cliff. A refcount would leak the 2-cycle; the arena bulk-drops it. This is the one place a Pile-B model earns a genuine new capability.
Second-class valuesinferred
/** Cyclic graph. Second-class values are SILENT here — a mutable cycle is exactly the
 *  aliased/escaping data the qualifier says nothing about. Surface identical to today. */
pub fn link(self, other: GNode) { self.next = other }   // field-store: `other` must be first-class

pub fn run() -> i64 {
    let a = GNode::make(1)
    let b = GNode::make(2)
    a.link(b)   // b stored into a.next ⇒ b is first-class
    b.link(a)   // a stored into b.next ⇒ a is first-class
    walk(a, 4)  // 1 + 2 + 1 + 2 = 6
}
Delta from arenaNone functionally. link stores other into a field, so a, b are inferred first-class — the qualifier correctly steps out of the way. start in walk inferred @local. Same code, same cap-bounded traversal.
LoweringIdentical to today: two arena objects, a back-edge set by field mutation, a step-capped walk. Qualifier erased. The cycle still lives on the arena, reclaimed only in bulk on region drop.
Honest flagDoes nothing for the actual cycle problem. Second-class values cannot reclaim a cycle, detect the leak, or prove isolation — a cyclic mutable graph is precisely the escaping/aliased data the model has zero power over. It infers a/b first-class and has nothing more to offer. The cap is still what makes the walk terminate.
Ordered typesinternal proof
/* Source byte-identical to arena S3 (mutual cycle a↔b set by mutation, capped walk).
 * INTERNAL PROOF OBLIGATION (invisible): ordered context Γ = [ a < b ].
 * The lens PROVES a and b are BOTH consumed on run()'s frame drop, later-first
 * (b ⊣ a), so BULK RECLAMATION is LIFO-sound: nothing outlives the frame.
 * But the CYCLE — the a.next=b / b.next=a back-edges — is invisible to ordered
 * logic: the fragment is about consumption ORDER, not the reachability GRAPH.
 * It proves 'the region drop is ordered-sound'; NOT 'the cycle is safe to walk' —
 * that safety comes entirely from the runtime step-cap, not the type system. */
Delta from arenaNothing (invisible). Ordered types certify only the frame-level LIFO drop of {a,b}; the cyclic aliasing is neither certified nor forbidden — simply outside the fragment's vocabulary.
LoweringIdentical: a and b bump-allocated in run()'s frame, both reclaimed in one region drop. The cycle needs no special reclamation — bulk region-drop ignores internal edges. Ordered types add no instruction.
Honest flagGives a false sense of coverage if oversold: they certify the drop is LIFO-ordered, but that was never the risky part — the risk is the cycle, and the fragment says nothing about aliasing or reachability. The traversal is safe only because of the runtime cap. A clean illustration that ordered logic addresses lifetime nesting, not the graph-shape cliffs that are Teko's actual hard problems.

Under REAPS the returned buffer is coalesced into the bump frontier and can be re-handed on the very next allocation, so the UAF through y races against reuse. Still no compile-time diagnostic.
Honest flag · makes it worseREAPS makes the aliased-free cliff worse, not better. Zero reachability/aliasing awareness (exactly the fact the whole frontier needs and REAPS does not supply), so it cannot diagnose the UAF; and faster recycling shortens the window before the freed memory is overwritten, converting stale-read bugs into corrupted-read bugs. Any safety story here must come from a different model; REAPS is orthogonal to safety entirely.
NOEMAreal syntax · the one real win
/** THE UAF PROBE — the one scenario the noema actually improves. */
pub fn run() -> i64 {
    mut x: []i64 = teko::list::empty()
    x = teko::list::push(x, 41); x = teko::list::push(x, 1)

    let y: &[]i64 = &x   // y is a NOEMA: read-only, tethered to x, may NOT outlive x
    teko::mem::free(x)   // COMPILE ERROR: cannot free/consume x while a live noema &x (y) borrows it

    y[0] + y[1]          // …so this read is never reached with a dangling y
}
/* If instead you wrote `let y = x` (an OWNING alias, no &), the model is silent — that is
   the hole it does NOT close: two owning names to one buffer is still unproven, UAF-capable. */
Delta from arenaThe alias is written as a noema: let y: &[]i64 = &x. In arena S6 let y = x is a silent owning alias and free(x) compiles into a UAF. Here, because y borrows x read-only, free(x) is rejected while the borrow is live — the UAF read is unreachable.
Lowering&x → const int* aliasing x's buffer; the borrow-liveness check runs at compile time only. When it passes, the freed-while-borrowed case simply never codegens; when you legitimately drop the borrow first, free lowers to today's free-list park. Zero runtime cost, no tag, no refcount.
Honest flag · the one real win, boundedThe model's one real win — and it is bounded. It fixes S6 only when the alias is a &T noema; the classic owning let y = x alias is exactly what the noema cannot see and does not protect. Worse, the soundness of "reject free while &x is live" depends on Teko proving its name-based escape/borrow tracking sees every live noema — a fact Neut gets from full linearity that Teko has not established. So the guarantee is real in shape but rests on unbuilt/unaudited analysis, not on the copied rule alone.
Relevant typesreal syntax · powerless here
pub fn run() -> i64 {
    #must_free mut x: []i64 = teko::list::empty()
    x = teko::list::push(x, 41); x = teko::list::push(x, 1)

    let y = x            // ALIAS. #must_free governs whether x is FREED, not whether an alias EXISTS.
                         // y is an ordinary borrow-shaped copy of the handle; the model does not track it.
    teko::mem::free(x)   // discharges x's obligation (good — no leak). But y still points at the freed buffer.

    y[0] + y[1]          // UNSAFE read through dangling alias — STILL not a compile error.
    // #must_free made the LEAK impossible; it did nothing about the USE-AFTER-FREE.
}
Delta from arenaOnly adds #must_free to the x binding. The free, the alias, and the unsafe read are byte-for-byte arena S6. The relevant type changes nothing about this scenario's outcome — included to show the boundary explicitly.
LoweringIdentical to arena-S6: push calls, one mem::free that scrubs the direct binding, a dangling read through y. The #must_free bit is erased and influences only leak-checking, which this program already passes (x IS freed). No effect on the UAF path.
Honest flag · powerless by constructionThe headline honest limit: relevant types are the dual of affine/linear — affine says "at most once," relevant says "at least once (must consume)." Neither is uniqueness. #must_free guarantees x is freed exactly on schedule, which is precisely why the UAF is now more reachable: the buffer is definitely gone by the time y reads it. Do not let leak-safety be read as memory-safety — S6 is still a live use-after-free.
Adoption & abandonreal syntax · Pile B · trap, not fix
/** The honest LIMIT of the model. To get any protection you must route the buffer THROUGH
 *  a region and use `abandon` instead of `free`. Then the alias makes the abandon-check TRAP.
 *  With plain teko::mem::free (no region), the model gives you NOTHING — same silent UAF. */
pub fn run() -> i64 {
    region r {
        let x = adopt(r, teko::list::of(41, 1))   // buffer adopted into region r
        let y = x                                   // ALIAS: y and x co-owned in r
        abandon(r, x)   // TRAPS at runtime: x is still aliased by y (dynamic per-object check)
                        // → use-after-free prevented by a RUNTIME trap, not a compile error
        y[0] + y[1]     // unreachable: abandon already trapped
    }
}
Delta from arenaArena S6 compiles and silently reads a dangling alias (UAF, caught only by ASan). The delta is conditional: IF you route the buffer through a region and reclaim with abandon, the mandatory dynamic per-object check traps on the surviving alias — turning silent UAF into a deterministic runtime abort. IF you keep plain teko::mem::free, the model changes nothing. It converts UAF-to-corruption into UAF-to-trap, but only inside the region discipline, and only as a runtime catch — never a compile error.
Loweringadopt(r, buf) tags the buffer with region id r. let y = x is a pointer copy; both refer to the same tagged buffer. abandon(r, x) lowers to the dynamic ownership guard: it detects the buffer still has a live co-owner (y) and calls the trap/panic path instead of returning storage — a runtime branch + abort, no static reasoning.
Honest flag · trap, not a closed cliffThe crucial honesty point: adoption-and-abandon does not make aliased-free statically safe — it does not compute the reachability/uniqueness fact Teko lacks. It only downgrades the failure from silent corruption to a deterministic runtime trap, and only when the buffer is inside a region and freed via abandon; plain free still UAFs silently. The check is dynamic (a runtime word + branch per abandon), fires at abandon-time not the aliasing site, and is Mezzo's post-mortemed mechanism. Net: better failure mode, not a closed cliff.

04  The strategic split

Pile A — steal a piece  ·  Pile B — bet the language

The five Pile-A models are cheap because they cannot regress a working program — they tighten an existing rule, prove an existing invariant, or optimise a runtime. The lone Pile-B model buys a genuine new capability (the cyclic-graph corner) but pays for it in runtime tags, a per-object dynamic check, and region-permission threading that is a discipline, not a free lens.

Pile A — steal a piece

Low-risk · mostly invisible · tightens or optimises, never regresses

  • Second-class values — inferred @local; a soundness floor under the existing Ref-can't-escape rule. The pilotable idea. Zero annotation, zero runtime cost; real cost is the escape-analysis audit.
  • Ordered types — invisible internal proof for the region tree. Certifies the easy 80%, honestly refuses the hard 20% (escape, aliasing, cycles).
  • NOEMA (&T) — one read-borrow rule; makes the read side of aliased-free sound. Bounded to borrow-shaped aliases; soundness leans on unbuilt tracking.
  • Relevant types (#must_free) — opt-in leak-catcher. Genuinely valuable at S2/S5; powerless on aliased-free by construction (the must-consume dual, not uniqueness).
  • REAPS — allocator swap, one reap { } scope. Footprint plateau for churn; touches no correctness cliff and worsens S6.

Pile B — bet the language

Real new power · real ongoing price · a discipline, not a lens

  • Adoption & abandon (Mezzo) — the only model that buys a capability the shipping arena cannot express: a sound, Coq-verified cyclic/aliased mutable graph in a bulk-reclaimed region, no refcount, no GC (S3/S4).
  • Paid in runtime — a per-object adoption tag word and a mandatory dynamic abandon-check; it sidesteps the reachability fact Teko lacks rather than computing it.
  • Regresses the easy case — S2 return-fresh becomes region-threading boilerplate or an O(n) abandon/re-adopt tax; the arena auto-promotes for free.
  • Leaves S6 open — abandon guards removal from a region; it downgrades aliased-free to a runtime trap, never a static fix, and only inside the discipline.
  • Full Mezzo was post-mortemed — the affordable version is a narrow library-plus-two-keywords hatch, not importing the permission calculus.

The honest bottom line

Nothing here closes the hard cliffs cheaply

Across all six models and all six scenarios, the pattern is consistent: the cheap ideas are cheap because they add almost nothing observable, and the one model that buys real new power (adoption & abandon) pays an ongoing runtime price, regresses the arena's easy case, and still leaves the aliased-free cliff open.

The pilotable idea
Second-class values. It is a soundness floor, not new power — an inferred @local bit that turns today's silent Ref-escape holes into compile errors and can only ever tighten an existing restriction, never regress a working program. Its real cost is the audit of Teko's name-based escape analysis, not the qualifier.
The invisible half
Ordered types (internal proof), REAPS (runtime allocator), and second-class values (inferred) put nothing in front of the programmer. "Has no surface" is the headline finding — it is exactly what makes them the low-risk Pile-A ideas, and exactly why none of them can reach the hard cliffs.
Every hard cliff, unmoved
Graph-leak, cycles (S3), aliased-free UAF (S6), and concurrency sit where they were. NOEMA closes only the borrow-shaped read side of S6; adoption downgrades S6 to a runtime trap inside a region; everything else is silent or, in REAPS' case, makes it worse.
What no model does
None of them computes the reachability/uniqueness fact the frontier actually needs. The models that touch S6 at all either extract one rule from a full-linearity discipline Teko refuses to adopt (NOEMA) or sidestep the fact with a runtime check (adoption). Sound static aliased-free remains Pile-B, bet-the-language research.

If a pilot ships, ship second-class values — precisely because it is the model most likely to change nothing observable while making the language more honest. Do not read any of these as a step toward memory safety on the hard cliffs; that step is not cheaply available here.
